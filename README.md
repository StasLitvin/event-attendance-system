# Event Attendance System

Система учёта посещений мероприятий по QR-кодам. Состоит из Telegram-бота для сканирования и Flask-приложения с веб-дашбордом и API. Данные хранятся в PostgreSQL.

> Этот проект сейчас лежит в корне репозитория `RES` вместе с другими. Рекомендуется вынести перечисленные ниже файлы в отдельную папку.

## Файлы проекта

- `mian.py` — точка входа: Telegram-бот (aiogram) + Flask-приложение, запускаются параллельно (бот в основном потоке, веб — в отдельном).
- `database.py` — подключение к PostgreSQL и модели SQLAlchemy: `User`, `Event`, `Participant`, `Attendance`, а также `init_db()` и `create_test_data()`.
- `templates/dashboard.html` — HTML-шаблон веб-дашборда.
- `requirements.txt` — зависимости.

## Возможности

- Регистрация событий и участников через веб-API; каждому участнику генерируется уникальный QR-код.
- Telegram-бот: выбор события, сканирование (приём текста QR-кода), отметка посещения с проверкой на повторный скан, личная статистика.
- Веб-дашборд: счётчики событий, участников и посещений, последние события.
- Генерация изображений QR-кодов на лету (`/api/qr/<code>`).
- Роли пользователей: `organizer` / `admin`.

## Технологии

aiogram 3, Flask, SQLAlchemy 2, PostgreSQL (psycopg2), qrcode[pil].

## Запуск

1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Поднимите PostgreSQL и задайте строку подключения (по умолчанию используется локальная БД `newdb`):
   ```bash
   export DATABASE_URL="postgresql+psycopg2://user:pass@localhost:5432/newdb?client_encoding=utf8"
   ```
3. Задайте токен бота и параметры веб-сервера через переменные окружения:
   ```bash
   export BOT_TOKEN="<ваш токен>"
   export WEB_HOST=0.0.0.0
   export WEB_PORT=8000
   ```
4. Запустите:
   ```bash
   python mian.py
   ```
   При старте инициализируется БД, создаются тестовые данные, поднимается веб-сервер и запускается бот.

## ⚠️ Безопасность

В текущей версии `mian.py` токен Telegram-бота **захардкожен в коде**. Обязательно отзовите этот токен в @BotFather, перевыпустите новый и храните его только в переменной окружения. То же касается строки подключения к БД с паролем.

## Замечания

- Имя файла `mian.py` — вероятно опечатка от `main.py`.
- В файле есть две функции `start_web()` (одна для Flask, одна с `uvicorn.run`); фактически используется Flask-вариант, а `uvicorn` не импортируется — дублирующую функцию стоит удалить.
